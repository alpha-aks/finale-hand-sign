\section{Dataset Challenge and Custom Solution}

\subsection{Problem: Dataset Unavailability}

The primary challenge faced during project development was the \textbf{unavailability of publicly available Indian Sign Language (ISL) datasets}. Publicly available sign language datasets are typically:

\begin{itemize}
    \item Limited to American Sign Language (ASL) or other international sign languages
    \item Restricted by licensing agreements
    \item Not freely accessible for commercial or research use
    \item Insufficient for training robust models
    \item Not tailored to specific recognition requirements
\end{itemize}

ISL-specific challenges:
\begin{itemize}
    \item Linguistic differences from ASL and other sign languages
    \item Region-specific gesture variations
    \item Limited online resources for ISL
    \item Lack of standardized ISL gesture libraries
\end{itemize}

\subsection{Solution: Custom Dataset Creation}

To overcome this challenge, we implemented a comprehensive custom dataset creation pipeline within the Streamlit dashboard:

\subsubsection{Dataset Collection Process}

\begin{enumerate}
    \item \textbf{Gesture Definition}: Define specific ISL gestures to recognize and their corresponding labels
    \item \textbf{Training Mode}: Use dashboard training interface to capture images
    \item \textbf{Data Collection}: Capture 100+ images per gesture from multiple angles
    \item \textbf{Data Augmentation}: Vary lighting, distance, and hand position
    \item \textbf{Labeling}: Automatic labeling through directory organization
    \item \textbf{Validation}: Manual verification of collected data
\end{enumerate}

\section{Complete System Pipeline}

The system pipeline consists of 7 major stages:

\subsection{Stage 1: Video Input and Preprocessing}

\paragraph{Input Layer}
\begin{itemize}
    \item Capture from webcam (can also process video files)
    \item Resolution: 640×480 pixels at 30 FPS
    \item Color format: BGR (OpenCV standard)
\end{itemize}

\paragraph{Preprocessing}
Frames are mirrored for intuitive interaction, converted from BGR to RGB, and wrapped in MediaPipe image format before landmark detection.

\paragraph{Output}
\begin{itemize}
    \item Normalized RGB image (0-1 range)
    \item Timestamp in milliseconds for video mode
\end{itemize}

\subsection{Stage 2: Hand Landmark Detection (MediaPipe)}

\paragraph{Hand Detection Configuration}
The hand landmarker is configured in video mode with dual-hand support and tuned detection/tracking confidence thresholds for stable real-time operation.

\paragraph{Landmark Structure}

MediaPipe provides 21 hand landmarks per hand:

\begin{table}[H]
\centering
\begin{tabular}{|c|c|c|}
\hline
\textbf{Index} & \textbf{Landmark} & \textbf{Position} \\
\hline
0 & Wrist & Base of hand \\
\hline
1-4 & Thumb & Base, middle, interphalangeal, tip \\
\hline
5-8 & Index & Base, PIP, DIP, tip \\
\hline
9-12 & Middle & Base, PIP, DIP, tip \\
\hline
13-16 & Ring & Base, PIP, DIP, tip \\
\hline
17-20 & Pinky & Base, PIP, DIP, tip \\
\hline
\end{tabular}
\caption{MediaPipe Hand Landmarks (21 points per hand)}
\end{table}

\paragraph{Output}
\begin{itemize}
    \item 21 landmarks per hand (x, y, z coordinates)
    \item Confidence scores (0-1)
    \item Handedness (LEFT/RIGHT)
    \item Detection results for multiple hands
\end{itemize}

\subsection{Stage 3: Landmark Preprocessing}

\paragraph{Normalization Process}

The raw landmarks require preprocessing before classification:
Landmarks are converted to pixel coordinates, wrist-centered, flattened to feature vectors, and normalized to improve robustness against translation and scale changes.

\paragraph{Preprocessing Strategy}
\begin{enumerate}
    \item \textbf{Wrist Centering}: Translate all points so wrist (index 0) is at origin
    \item \textbf{Flattening}: Convert 21 (x,y) pairs into 42-element vector
    \item \textbf{Normalization}: Divide by maximum value to scale to [-1, 1]
    \item \textbf{Invariance}: Makes gesture recognition scale and position invariant
\end{enumerate}

\subsection{Stage 4: Gesture Classification}

\paragraph{Keypoint Classifier Architecture}
The classifier uses dense layers with dropout regularization and softmax output, optimized with Adam for multi-class landmark-based gesture prediction.

\paragraph{Classification Process}

\begin{enumerate}
    \item Preprocessed landmarks (42 values) → Input layer
    \item Dense layer 1: 42 → 128 neurons with ReLU activation
    \item Dropout: Random 20\% deactivation to prevent overfitting
    \item Dense layer 2: 128 → 64 neurons with ReLU activation
    \item Dropout: 20\% to improve generalization
    \item Output layer: 64 → Number of gesture classes with Softmax
    \item Result: Probability distribution over gestures
\end{enumerate}

\paragraph{Decision Making}
Predicted class confidence is checked against a threshold before accepting and processing a gesture to reduce false positives.

\subsection{Stage 5: Point History Analysis (Complex Gestures)}

\paragraph{Purpose}

Recognize gestures involving motion (swipes, circles) using trajectory history:
A fixed-size history buffer stores fingertip trajectory points so motion-based patterns can be distinguished from static signs.

\paragraph{Point History Preprocessing}
Trajectory coordinates are origin-shifted, normalized by frame size, and flattened into temporal feature vectors for sequence classification.

\paragraph{Output}
\begin{itemize}
    \item 32-element feature vector (16 frames × 2 coordinates)
    \item Trajectory-based gesture classification
    \item Used for detecting motion-based gestures
\end{itemize}

\subsection{Stage 6: Text Output Generation}

\paragraph{Label Lookup}
Predicted class indices are mapped to gesture text using CSV label files to generate user-readable output.

\paragraph{Output Management}
\begin{itemize}
    \item Gesture text stored in variable
    \item Confidence score calculated
    \item Duplicate detection (skip if same as last)
    \item CSV logging for training data collection
\end{itemize}

\subsection{Stage 7: Speech Synthesis and Display}

\paragraph{Text-to-Speech Conversion}
Speech output is executed asynchronously through a background worker so recognition remains responsive during audio playback.

\paragraph{Display Output}
The UI layer overlays hand type, gesture label, and status annotations on frames for immediate visual feedback.

\subsection{Snippet 3: TFLite Gesture Prediction (keypoint\_classifier.py)}

\begin{lstlisting}[language=Python]
class KeyPointClassifier(object):
    def __init__(self, model_path='model/keypoint_classifier/keypoint_classifier.tflite'):
        self.interpreter = tf.lite.Interpreter(model_path=model_path, num_threads=1)
        self.interpreter.allocate_tensors()
        self.input_details = self.interpreter.get_input_details()
        self.output_details = self.interpreter.get_output_details()

    def __call__(self, landmark_list):
        self.interpreter.set_tensor(
            self.input_details[0]['index'],
            np.array([landmark_list], dtype=np.float32)
        )
        self.interpreter.invoke()
        result = self.interpreter.get_tensor(self.output_details[0]['index'])
        return np.argmax(np.squeeze(result))
\end{lstlisting}

\subsection{Snippet 4: Landmark Preprocessing and Normalization (app.py)}

\begin{lstlisting}[language=Python]
def calc_landmark_list(image, landmarks):
    image_width, image_height = image.shape[1], image.shape[0]
    landmark_point = []
    for _, landmark in enumerate(landmarks.landmark):
        landmark_x = min(int(landmark.x * image_width), image_width - 1)
        landmark_y = min(int(landmark.y * image_height), image_height - 1)
        landmark_point.append([landmark_x, landmark_y])
    
    return landmark_point

def pre_process_landmark(landmark_list):
    base_x, base_y = landmark_list[0]
    landmark_list = np.array(landmark_list)
    landmark_list[:, 0] = landmark_list[:, 0] - base_x
    landmark_list[:, 1] = landmark_list[:, 1] - base_y
    
    max_value = np.max(np.abs(landmark_list))
    if max_value > 0:
        landmark_list = landmark_list / max_value
    
    return landmark_list.flatten().tolist()
\end{lstlisting}

\section{Model Training and Deployment}

\subsection{Training Pipeline}

The model training process includes:
\begin{enumerate}
    \item Load collected training data from CSV files
    \item Normalize input features (zero mean, unit variance)
    \item Split data into training (80\%) and validation (20\%) sets
    \item Initialize Keras Sequential model with specified architecture
    \item Train model using Adam optimizer and categorical cross-entropy loss
    \item Monitor validation accuracy and apply early stopping if needed
    \item Save best model in both HDF5 and TFLite formats
\end{enumerate}

\subsection{Model Architecture}

The gesture classification model uses the following layers:
\begin{itemize}
    \item \textbf{Input}: 42 features (21 landmarks × 2 coordinates)
    \item \textbf{Dense 128}: ReLU activation with 128 units
    \item \textbf{Dropout}: 20\% dropout for regularization
    \item \textbf{Dense 64}: ReLU activation with 64 units
    \item \textbf{Dropout}: 20\% dropout for regularization
    \item \textbf{Dense 32}: ReLU activation with 32 units
    \item \textbf{Output}: Softmax with number-of-gestures units
\end{itemize}

\subsection{Model Conversion and Optimization}

After training, the model is converted to TensorFlow Lite for deployment:
\begin{enumerate}
    \item Export trained Keras model to TFLite format
    \item Apply quantization to reduce model size (60\% reduction typical)
    \item Validate TFLite model accuracy on test set
    \item Deploy TFLite model to model/keypoint\_classifier/ directory
    \item Retain HDF5 version for future retraining
\end{enumerate}

\section{Runtime Processing Pipeline}

\subsection{Frame Processing Loop}

The main recognition loop executes the following steps per frame:

\begin{enumerate}
    \item Capture raw frame from webcam (BGR format)
    \item Flip frame horizontally for mirror effect
    \item Convert BGR to RGB for MediaPipe compatibility
    \item Run MediaPipe HandLandmarker to detect hand landmarks
    \item For each detected hand:
    \begin{enumerate}
        \item Extract 21 hand landmarks with confidence scores
        \item Filter landmarks with confidence < threshold
        \item Preprocess landmarks (normalize, scale)
        \item Add to fixed-size history buffer
        \item Run keypoint classifier on current landmarks
        \item If history sufficient, run point history classifier
        \item Map class index to gesture name using label CSV
        \item Check confidence threshold before reporting
    \end{enumerate}
    \item Combine results from both hands (if present)
    \item Update gesture history log
    \item Display annotated frame with landmarks and results
    \item Generate speech output if new gesture detected
\end{enumerate}

\subsection{Confidence Handling and Thresholding}

\begin{itemize}
    \item \textbf{Landmark Confidence}: Only landmarks with confidence > 0.5 used
    \item \textbf{Classification Confidence}: Predictions with confidence > 0.7 reported
    \item \textbf{Repeated Recognition}: Same gesture confirmed for 5+ consecutive frames
    \item \textbf{Gesture Change}: New gesture triggers only on confidence increase
    \item \textbf{Ambiguous Cases}: Confidence range 0.5-0.7 shown with uncertainty indicator
\end{itemize}

\section{Speech Integration and Output}

\subsection{Text-to-Speech Implementation}

Speech synthesis is implemented using pyttsx3 in a background thread:
\begin{enumerate}
    \item Recognized gesture text placed in output queue
    \item Background thread monitors queue continuously
    \item When text available:
    \begin{enumerate}
        \item Set speech rate (default 175 WPM)
        \item Queue text for synthesis
        \item Initiate non-blocking playback
    \end{enumerate}
    \item Thread continues monitoring for new text while previous speech plays
    \item No UI blocking even during long speech synthesis
\end{enumerate}

\subsection{Audio Configuration}

\begin{itemize}
    \item \textbf{Engine}: pyttsx3 with system default voice
    \item \textbf{Rate}: 150-200 WPM (configurable)
    \item \textbf{Volume}: 100\% (system volume controls final output)
    \item \textbf{Voice}: System default (varies by OS)
    \item \textbf{Threading}: Non-blocking background thread
\end{itemize}

\subsection{Common Issues and Solutions}

\begin{itemize}
    \item \textbf{Camera Not Detected}: Check device index and permissions
    \item \textbf{Low FPS}: Reduce frame resolution or simplify preprocessing
    \item \textbf{Poor Recognition}: Collect more training data, improve lighting
    \item \textbf{Speech Not Playing}: Check system audio settings and volume
    \item \textbf{Model File Missing}: Verify file paths and reinstall models
\end{itemize}

\subsection{Performance Tuning}

For different hardware:
\begin{itemize}
    \item \textbf{Low-End Hardware}: Reduce frame resolution, skip validation
    \item \textbf{Standard Hardware}: Default settings (640x480 @ 30 FPS)
    \item \textbf{High-End Hardware}: Increase resolution, parallel hand processing
    \item \textbf{Embedded Systems}: Use more aggressive quantization
\end{itemize}
